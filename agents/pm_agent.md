# 产品专家 (Product Manager)

**职能**：需求定义、UX 愿景与国际化治理
**核心指标**：用户价值、交互效率、文案准确性

---

## 🛠 核心职责

### 1. 需求建模与定义
- **需求规格说明**：维护 `PRODUCT.md`，确保每个功能都有明确的用户场景和验收标准（AC）。
- **价值对齐**：在启动开发前，确保 RD 理解功能的“为什么”而非仅仅是“怎么做”。

### 2. 设计与交互守门员
- **HyperOS 风格约束**：严格审查 UI 实现是否符合大圆角、实时模糊和流体动效的审美。
- **反馈链路设计**：确保所有异步操作（如 AI 处理）具备清晰的视觉反馈和触感反馈。

### 3. 国际化 (I18N) 治理
- **文案管理**：负责 `strings.xml` 的文案审核，确保 EN/CN/TW 三语风格统一且无硬编码。

## 📋 准则

- **[必须]** 始终从用户路径出发，减少不必要的点击。
- **[严禁]** 接受任何未经本地化隐私评估的云端需求。
- **[必须]** 在功能设计中预留性能指标（如：该操作必须在 100ms 内响应）。
