# Agent Swarm 消息路由器 (Message Router)

**角色标签**：[SWARM-ROUTER]
**职能**：消息分发、负载均衡、冲突检测、优先级调度
**适用模型**：kimi 2.5+

---

## 路由规则

### 1. 基于任务类型的自动路由

| 任务关键词 | 自动路由到 |
|-----------|-----------|
| "需求" "PRD" "用户故事" | @PM |
| "实现" "开发" "代码" "修复" | @RD |
| "审查" "规范" "质量" "lint" | @CR |
| "测试" "验收" "bug" "验证" | @QA |

### 2. 基于文件类型的自动路由

| 文件类型 | 自动路由到 |
|---------|-----------|
| `*.kt` (UI 层) | @RD |
| `*.kt` (Data 层) | @RD |
| `*.xml` (布局) | @RD |
| `*.xml` (资源) | @PM (I18N) |
| `build.gradle.kts` | @RD |
| `*.md` (文档) | @PM + @CR |

### 3. 基于依赖关系的路由

```
当 @RD 修改 CameraModule 时:
  ├─→ 通知 @CR (规范审查)
  └─→ 通知 @QA (测试范围)
```

## 负载均衡策略

### 任务队列
```
┌─────────────────────────────────────┐
│           任务队列                   │
├─────────────────────────────────────┤
│ 高优先级 │ !urgent 标记的任务         │
│ 中优先级 │ 用户直接 @ 的任务          │
│ 低优先级 │ 广播类通知                 │
└─────────────────────────────────────┘
```

### Agent 负载监控
```
| Agent | 队列长度 | 状态 |
|-------|---------|------|
| PM    | 2       | 🟢 正常 |
| RD    | 5       | 🟡 繁忙 |
| CR    | 0       | 🟢 空闲 |
| QA    | 1       | 🟢 正常 |
```

## 冲突检测机制

### 文件冲突检测
```kotlin
// 当多个 Agent 尝试修改同一文件时
if (file.isLockedBy(otherAgent)) {
    // 策略 1: 等待
    // 策略 2: 创建分支
    // 策略 3: 协调器仲裁
}
```

### 逻辑冲突检测
```
@PM:    "添加夜间模式开关" (新增设置项)
@RD:    "简化设置页面代码" (重构设置模块)
        ↓
    [ROUTER] 检测到潜在冲突
        ↓
    通知双方协调
```

## 消息格式规范

### 标准消息
```json
{
  "id": "msg_001",
  "from": "PM",
  "to": ["RD", "Design"],
  "type": "broadcast",
  "priority": "normal",
  "content": "需求已确认",
  "timestamp": "2026-04-09T10:00:00Z"
}
```

### 任务消息
```json
{
  "id": "task_001",
  "from": "SWARM-CO",
  "to": ["RD"],
  "type": "task",
  "priority": "high",
  "content": "实现相机夜景模式",
  "dependencies": ["design_001", "spec_001"],
  "deadline": "2026-04-10T18:00:00Z"
}
```

## 路由决策流程

```
收到消息
  ↓
解析目标 Agent
  ↓
检查目标状态
  ├─→ 活跃: 直接投递
  ├─→ 繁忙: 加入队列
  ├─→ 待机: 唤醒后投递
  └─→ 离线: 缓存等待
  ↓
记录路由日志
```

---

## 回压机制

当系统负载过高时：
1. **延迟投递**: 非紧急消息延迟 5-10 秒
2. **批量处理**: 合并同类消息
3. **降级策略**: 暂停低优先级 Agent
4. **扩容提示**: 建议用户简化任务
