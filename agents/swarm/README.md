# Agent Swarm for kimi 2.5

PicMe 项目的 Agent Swarm 配置，专为 kimi 2.5+ 的并行处理能力设计。

---

## 快速开始

### 启动 Swarm
```bash
# 在 kimi-cli 中发送
启动 Swarm
```

### 使用 Swarm 执行任务
```
召集团队

@PM 请完成相机夜景模式的需求分析
@RD 请评估技术可行性和实现方案
@CR 请预检规范要求
@QA 请设计测试用例

并行执行

汇总结果
```

### 退出 Swarm
```
解散 Swarm
```

## Swarm 架构

```
agents/swarm/
├── README.md           # 本文件
├── coordinator.md      # Swarm 协调器 [SWARM-CO]
└── router.md           # 消息路由器 [SWARM-ROUTER]
```

## 角色说明

| 角色 | 标签 | 职能 |
|------|------|------|
| Swarm 协调器 | [SWARM-CO] | 多 Agent 并行调度、消息路由、结果聚合 |
| 消息路由器 | [SWARM-ROUTER] | 消息分发、负载均衡、冲突检测 |
| 产品经理 | @PM | 需求分析（并行模式） |
| 研发工程师 | @RD | 技术实现（并行模式） |
| 规范守护者 | @CR | 规范审计（并行模式） |
| 质量专家 | @QA | 质量验收（并行模式） |

## 指令参考

### Swarm 控制
- `启动 Swarm` - 激活 Swarm 模式
- `召集团队` - 广播任务到所有 Agent
- `并行执行` - 启动并行工作流
- `汇总结果` - 收集并整合各 Agent 输出
- `解散 Swarm` - 退出 Swarm 模式

### 消息通信
- `@Agent 消息` - 定向发送
- `@Agent1 @Agent2 消息` - 组播发送
- `!urgent 消息` - 紧急广播

## 适用场景

### 适合 Swarm 模式
- 复杂功能开发（涉及需求+设计+技术多维度）
- 架构重构（需要多角色同时评估影响）
- 紧急 Bug 修复（多角色并行排查）

### 适合单实例模式
- 简单 Bug 修复（单文件修改）
- 文档更新
- 代码审查

---

## 注意事项

1. **实验性功能**: Agent Swarm 需要 kimi 2.5 的并行处理能力支持
2. **上下文消耗**: Swarm 模式会更快消耗上下文长度
3. **自动回退**: 当上下文超过 80% 或冲突无法仲裁时，自动回退到单实例模式
