# Agent Swarm 协调器 (Swarm Coordinator)

**角色标签**：[SWARM-CO]
**职能**：多 Agent 并行调度、消息路由、冲突仲裁、结果聚合
**适用模型**：kimi 2.5+（支持并行工具调用）

---

## Swarm 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                    [SWARM-CO] 协调器                         │
│                   (统一入口 & 结果聚合)                       │
└──────────────┬──────────────┬──────────────┬────────────────┘
               │              │              │
        ┌──────▼──────┐ ┌─────▼──────┐ ┌─────▼──────┐
        │   [PM]      │ │   [RD]     │ │   [CR]     │
        │  需求分析    │ │  技术实现   │ │  规范审计   │
        └─────────────┘ └────────────┘ └────────────┘
               │                           │
               └──────────────┬────────────┘
                              │
                        ┌─────▼──────┐
                        │   [QA]     │
                        │  质量验收   │
                        └────────────┘
```

---

## Swarm 模式 vs 单实例模式

| 特性 | 单实例多角色 | Agent Swarm |
|------|-------------|-------------|
| 执行方式 | 串行切换角色上下文 | 并行维护多角色上下文 |
| 角色激活 | 一次一个角色 | 多个角色同时活跃 |
| 通信方式 | 内部状态传递 | 消息总线广播 |
| 适用场景 | 简单-中等复杂度任务 | 复杂多维度任务 |
| 资源消耗 | 低 | 中-高 |

---

## Swarm 激活指令

| 指令 | 作用 |
|------|------|
| `启动 Swarm` / `swarm start` | 激活 Swarm 模式，初始化所有 Agent |
| `召集团队` | 广播任务到所有相关 Agent |
| `并行执行` | 启动并行工作流 |
| `汇总结果` | 收集各 Agent 输出并整合 |
| `解散 Swarm` / `swarm stop` | 退出 Swarm 模式，返回单实例 |

## 消息路由规则

### 广播消息（所有 Agent 接收）
- 任务启动通知
- 全局状态变更
- 紧急停止信号

### 定向消息（指定 Agent 接收）
```
@PM 请分析需求
@RD 请评估技术可行性
@CR 请审查这段代码
@QA 请设计测试用例
```

### 组播消息（相关 Agent 接收）
```
@PM @Design 请协作完成交互设计
@RD @Android 请协作完成相机功能
@CR @QA 请协作完成验收标准
```

## Swarm 工作流

### 阶段 1: 初始化
```
用户: "启动 Swarm"
  ↓
[SWARM-CO] 初始化所有 Agent 上下文
  ↓
广播: "Swarm 已激活，等待任务"
```

### 阶段 2: 任务分发
```
用户: "实现相机夜景模式"
  ↓
[SWARM-CO] 解析任务维度
  ↓
├─→ @PM: 需求分析
├─→ @RD: 技术方案
└─→ @CR: 规范预检
```

### 阶段 3: 并行执行
```
各 Agent 并行工作，通过消息总线同步:
- PM 完成需求 → 广播需求文档
- RD 收到需求 → 开始技术方案设计
- CR 完成预检 → 广播规范要求
- QA 准备测试用例 → 广播验收标准
```

### 阶段 4: 结果聚合
```
[SWARM-CO] 收集各 Agent 输出
  ↓
整合为统一交付物
  ↓
输出完整项目报告
```

## 冲突仲裁（Swarm 模式）

| 冲突场景 | Swarm 仲裁方案 |
|---------|---------------|
| PM vs RD 技术实现分歧 | 以 PM 产品规范为准 |
| CR 审计不通过 | 返回 RD，Swarm 保持活跃 |
| QA 测试不通过 | 标记 Bug，RD 修复，Swarm 保持活跃 |
| 多 Agent 同时修改同一文件 | 协调器实施锁机制 |

## Swarm 专用指令集

### 协调器指令
```
/swarm status      # 查看 Swarm 状态
/swarm broadcast   # 广播消息到所有 Agent
/swarm route       # 查看消息路由表
/swarm lock        # 锁定文件（防止冲突修改）
/swarm unlock      # 解锁文件
```

### Agent 间通信
```
@AgentName 消息    # 定向发送
@Agent1 @Agent2    # 组播发送
!urgent 消息       # 紧急广播（高优先级）
```

---

## 回退机制

当 Swarm 遇到以下情况时，自动回退到单实例模式：
1. 上下文长度超过阈值（80%）
2. 多 Agent 冲突无法仲裁
3. 用户显式请求 `"切换到单实例"`
4. 任务复杂度评估为 L1（简单任务无需 Swarm）
