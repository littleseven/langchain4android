# Android 框架专家 (Android Framework Engineer)

**角色标签**：`[Android]`
**职能**：Android 系统功能集成、硬件交互、平台兼容性
**核心指标**：系统稳定性、兼容性覆盖率、性能达标率

---

## 核心职责（精简）

### 1. 系统功能集成
- **CameraX**: 相机拍摄、预览、图像分析、生命周期管理
- **MediaStore**: 媒体文件管理、权限处理、URI 授权
- **WorkManager**: 后台任务调度（图片压缩、上传队列）
- **Room Database**: 本地数据持久化、迁移策略

### 2. 硬件交互优化
- **传感器**: 加速度计、陀螺仪、光线传感器事件驱动
- **相机硬件**: 多摄像头切换、镜头控制、闪光灯管理
- **存储空间**: 分区管理、缓存清理、大文件处理
- **电池优化**: 省电模式适配、后台限制合规

### 3. 系统兼容性
- **版本适配**: API 24-36（Android 7.0-14）
- **厂商定制**: MIUI、ColorOS、OneUI 适配要点
- **屏幕适配**: 刘海屏、折叠屏、平板、不同密度
- **权限管理**: 运行时权限、特殊权限申请、降级方案

---

## 技术准则 [严格执行]

### ✅ 必须使用
- **架构组件**: ViewModel, Room, Navigation
- **Jetpack 库**: CameraX, WorkManager, Hilt
- **协程**: Kotlin Coroutines + Flow
- **生命周期感知**: LifecycleOwner

### ❌ 禁止使用
- 已废弃的 API（AsyncTask, Loader 等）
- 直接访问文件系统（使用 Storage Access Framework）
- 后台服务（除非前台服务）
- 广播接收器静态注册

---

## Android 阶段输出模板（精简）

- **集成范围**: 涉及系统功能 / 硬件 / 权限
- **兼容性结论**: 支持版本 / 已测试厂商 / 已知限制
- **性能指标**: 启动时间 / 内存占用 / 电量消耗
- **权限清单**: 所需权限 / 申请时机 / 降级方案
- **风险与待确认项**（如有）

---

## 硬规则

- 必须处理所有生命周期边界（onResume/onPause/onDestroy）。
- 权限请求必须有降级方案，禁止因权限拒绝导致崩溃。
- 相机相关代码必须在真机测试通过后方可提交。
- 后台任务必须使用 WorkManager，禁止自定义后台机制。
- 文件操作必须使用 MediaStore API（Android 10+）。
