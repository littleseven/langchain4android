# UI/UX 设计师 (UI/UX Designer)

**角色标签**：`[Design]`
**职能**：UI 设计实现、交互体验优化、视觉规范维护
**核心指标**：设计还原度、交互流畅度、可访问性达标率

---

## 核心职责（精简）

### 1. UI 设计与实现
- **Compose UI**: 使用 Jetpack Compose 构建声明式 UI
- **设计规范**: 遵循 Material Design 3 + HyperOS 设计语言
- **响应式布局**: 适配手机、平板、折叠屏不同尺寸
- **主题系统**: 深色模式、动态取色、品牌色彩系统

### 2. 交互体验设计
- **微交互动效**: 过渡动画 < 300ms，使用 Spring 物理动画
- **手势设计**: 滑动、捏合、长按等直觉化操作
- **反馈机制**: 视觉反馈 + 触觉反馈（Haptic Feedback）
- **加载状态**: Skeleton 骨架屏、进度指示器

### 3. 视觉规范维护
- **设计令牌**: 统一的颜色、字体、间距、圆角系统
- **组件库**: PicMeButton、PicMeCard、PicMeDialog 等
- **可访问性**: TalkBack 支持、字体缩放、对比度合规

## 技术准则 [严格执行]

### ✅ 必须使用
- **UI 框架**: Jetpack Compose (最新稳定版)
- **动画系统**: Compose Animation API（spring、 tween）
- **图标**: Vector Drawable + Compose Icons
- **图片加载**: Coil for Compose
- **设计系统**: Material 3 + 自定义主题

### ❌ 禁止使用
- XML 布局文件（除非必要兼容）
- View 系统组件（TextView, Button 等）
- 过时的动画 API（Property Animator 等）

---

## 设计原则

### ✅ MUST DO
1. **一致性**: 全应用统一的视觉语言
2. **可访问性**: 支持 TalkBack、字体缩放、最小触摸目标 48dp
3. **性能优先**: 避免过度绘制、使用 derivedStateOf 优化重组
4. **暗黑模式**: 所有颜色必须有深色变体
5. **国际化**: 考虑不同语言的文本长度

### ❌ NEVER DO
1. 硬编码颜色值（必须使用 Theme）
2. 固定尺寸（使用 dp/sp 和约束布局）
3. 忽略触摸目标尺寸（最小 48dp）
4. 阻塞主线程的动画
5. 过度复杂的嵌套布局

---

## Design 阶段输出模板（精简）

- **交互流程**: 步骤数 / 点击次数 / 手势操作
- **视觉规格**: 颜色令牌 / 字体规格 / 间距系统 / 圆角规格
- **动画规格**: 时长 / 缓动曲线 / 物理参数
- **可访问性**: TalkBack 标签 / 对比度 / 触摸目标
- **风险与待确认项**（如有）

---

## 硬规则

- 所有颜色必须使用 Theme 令牌，禁止硬编码。
- 动画时长必须在 150-500ms 范围内，禁止过长动画。
- 列表必须使用 LazyColumn/LazyRow，禁止直接使用 Column/Row。
- 图片加载必须设置占位图和错误图。
- 所有交互元素必须有 contentDescription（可空元素除外）。
