## 战略总览

PicMe 是一个 **"AI 影像技术研究项目"**，以技术探索与能力验证为核心目标，三条产品线并行推进：

| 产品线 | 定位 | 核心价值 | 研究目标 |
|--------|------|----------|----------|
| **拍照** | 能力验证基线 | 零延迟 + 自然美颜 + 隐私安全 | 实时渲染管线、人脸关键点驱动美颜 |
| **视频** | 技术挑战场景 | 实时美颜录制 + 帧同步稳定 | GPU 管线在录制场景的极限验证 |
| **相册及编辑** | 算法效果放大器 | 本地 AI 修图 + 精准局部控制 | 美颜算法离线效果评估、编辑体验研究 |

**三条线的关系**：
```
拍照（能力基线）→ 视频（极限挑战）→ 编辑（效果评估）
      ↓                  ↓                  ↓
  大美丽引擎统一支撑（美颜/滤镜/妆容/关键点）
```

> **项目性质**：本项目为技术研究项目，暂无商业化与收费计划。所有功能免费开放，核心目标是沉淀影像算法能力与 AI 协作研发方法论。

## 1. 拍照线（Camera）

### 1.1 产品定位
**核心体验底座**。拍照是用户首次打开 PicMe 的默认场景，也是大美丽引擎能力的最直接展示窗口。

### 1.2 核心功能矩阵

| 功能 | 状态 | 优先级 | 说明 |
|------|------|--------|------|
| 实时美颜预览 | ✅ 已落地 | P0 | 磨皮/美白/瘦脸/大眼/唇色/腮红，106点驱动 |
| 零延迟快门 | ✅ 已落地 | P0 | < 50ms 快门延迟，三位反馈 |
| 场景识别 | ✅ 已落地 | P0 | 夜景/月亮自动触发 |
| 风格特效 | ✅ 已落地 | P0 | 卡通/素描/浮雕/色块/交叉线 |
| 帧同步美妆 | 🔄 进行中 | P0 | 解决妆容甩飞，支撑视频美颜 |
| **智能场景模板** | 📋 规划中 | P0 | 一键切换「清透日常/精致约会/酷飒街拍」 |
| 证件照模式 | 📋 规划中 | P1 | 自动抠图换背景、美颜降级 |
| 连拍优选 | 📋 规划中 | P1 | 表情识别自动标记最佳照片 |
| RAW 格式支持 | 📋 规划中 | P2 | 专业摄影用户 |

### 1.3 演进路线

**Phase 1（近期，2-4 周）：体验夯实**
- 帧同步美妆系统落地（P0）
- 人脸检测引擎收敛：InsightFace 为主，MediaPipe 降级为纯回退
- 拍照后处理 GPU 化性能优化（1080p < 200ms）

**Phase 2（中期，4-8 周）：智能升级**
- 智能场景模板上线（5 个预设场景）
- 美颜参数记忆功能（记住用户偏好）
- 笑脸快门、侧脸降强度等智能辅助

**Phase 3（长期，8-16 周）：场景扩展**
- 证件照模式（需 Selfie Segmentation）
- 连拍优选（基于表情检测）
- RAW 格式支持

### 1.4 验收标准

| 指标 | 红线 | 目标 |
|------|------|------|
| 冷启动首帧 | ≤ 500ms | ≤ 400ms |
| 快门延迟 | ≤ 50ms | ≤ 30ms |
| 参数跟手性 | ≤ 100ms | ≤ 50ms |
| 帧同步偏差（快转头）| ≤ 16px @1080p | ≤ 8px @1080p |
| 拍照成功率 | 100% | 100%（GPU 失败回退 CPU） |

## 2. 视频线（Video）

### 2.1 产品定位
**技术挑战场景**。视频录制对 GPU 管线性能、帧同步稳定性、实时美颜一致性提出更高要求，是验证大美丽引擎极限能力的最佳场景。

### 2.2 核心功能矩阵

| 功能 | 状态 | 优先级 | 说明 |
|------|------|--------|------|
| **基础视频录制** | 📋 规划中 | P0 | 720p/1080p，15s/60s/3min 三档时长 |
| **实时美颜录制** | 📋 规划中 | P0 | 录制过程中实时应用大美丽美颜 |
| **帧同步美妆（视频）** | 📋 规划中 | P0 | 录制时妆容不甩飞、不跳变 |
| 智能运镜模板 | 📋 规划中 | P1 | 提供「推拉摇移」模板，用户跟拍 |
| 分段录制 | 📋 规划中 | P1 | 多段视频拼接，支持回退重录 |
| 视频滤镜/特效 | 📋 规划中 | P1 | 实时风格特效录制 |
| Vlog 模板 | 📋 规划中 | P2 | BGM + 转场 + 字幕一键生成 |
| 视频导出优化 | 📋 规划中 | P2 | 适配抖音 9:16、小红书 3:4、朋友圈 1:1 |

### 2.3 演进路线

**Phase 1（近期，2-4 周）：基础能力**
- MediaCodec 录制管线接入
- 720p/1080p 基础录制（无时长的简单录制）
- 录制时复用现有预览管线（无美颜或仅基础美颜）

**Phase 2（中期，4-8 周）：美颜录制**
- 帧同步美妆系统接入视频录制链路
- 1080p@30fps 实时美颜录制（不丢帧）
- 分段录制 + 片段管理

**Phase 3（长期，8-16 周）：智能创作**
- 智能运镜模板（5-10 个模板）
- Vlog 一键生成（BGM + 转场 + 字幕）
- 多平台导出格式适配

### 2.4 验收标准

| 指标 | 红线 | 目标 |
|------|------|------|
| 录制帧率 | ≥ 30fps | ≥ 30fps（1080p 不丢帧） |
| 录制美颜一致性 | ≥ 95% | ≥ 99%（预览与录制效果一致） |
| 帧同步偏差（录制）| ≤ 16px @1080p | ≤ 8px @1080p |
| 视频导出速度 | ≤ 2x 时长 | ≤ 1x 时长 |
| 录制稳定性 | 崩溃率 ≤ 0.1% | ≤ 0.05% |

### 2.5 关键技术依赖

```
视频录制能力依赖链：
帧同步美妆系统（P0）→ 视频美颜不甩飞
  ↓
MediaCodec 录制管线（P0）→ 视频编码
  ↓
GPU 管线性能优化（P1）→ 1080p@30fps 不丢帧
```

## 3. 相册及编辑线（Gallery & Editor）

### 3.1 产品定位
**算法效果评估与放大器**。离线编辑场景可以脱离实时性能约束，更纯粹地验证美颜算法的上限效果，也是用户检验编辑成果的核心场景。

### 3.2 核心功能矩阵

| 功能 | 状态 | 优先级 | 说明 |
|------|------|--------|------|
| 相册浏览 | ✅ 已落地 | P0 | 时间轴 + 缩略图，120fps 滑动 |
| 静态图美颜编辑 | ✅ 已落地 | P0 | 复用相机美颜管线，GPU 离屏渲染 |
| 智能分组 | ✅ 已落地 | P1 | 人脸聚类 + 场景标签 |
| **AI 一键优化** | 📋 规划中 | P0 | 智能识别场景，一键推荐参数 |
| **精准局部美颜** | 📋 规划中 | P0 | 左眼/右眼/左脸/右脸 独立调节 |
| **智能消除** | 📋 规划中 | P1 | 圈选物体自动消除，AI 填充 |
| 专业调色面板 | 📋 规划中 | P1 | 曲线、HSL、分区调色 |
| 批量编辑 | 📋 规划中 | P1 | 多图批量应用同一套参数 |
| 证件照制作 | 📋 规划中 | P2 | 自动抠图换背景、合规检测 |
| 拼图/排版 | 📋 规划中 | P2 | 多图拼接、创意排版 |

### 3.3 演进路线

**Phase 1（近期，2-4 周）：智能修图**
- AI 一键优化：自动识别场景（人像/风景/美食/文档），一键应用参数
- 精准局部美颜：基于 106 点关键点，支持面部/身体部位独立调节
- 编辑体验优化：参数调节实时预览、历史记录、撤销/重做

**Phase 2（中期，4-8 周）：专业工具**
- 智能消除：基于本地轻量 Inpainting 模型
- 专业调色面板：曲线、HSL、色调分离
- 批量编辑：多图批量处理
- 导出优化：多格式/多尺寸导出

**Phase 3（长期，8-16 周）：商业化**
- 证件照制作：自动抠图 + 背景替换 + 美颜降级
- 拼图/排版：模板化拼图
- Pro 版功能解锁：高级功能一次性付费

### 3.4 验收标准

| 指标 | 红线 | 目标 |
|------|------|------|
| 编辑预览延迟 | ≤ 350ms | ≤ 200ms |
| 保存为新文件 | 100% 不覆盖原图 | 100% |
| AI 分析耗时 | ≤ 1s | ≤ 500ms |
| 批量处理速度 | ≤ 2s/张 | ≤ 1s/张 |
| 编辑保存率 | ≥ 40% | ≥ 60% |

## 4. 里程碑规划

### Phase 1（近期，2-4 周）：基础能力夯实

**目标**：三条线核心链路跑通，体验达到可用标准。

| 产品线 | 关键交付 |
|--------|----------|
| 拍照 | 帧同步美妆系统落地；引擎收敛（InsightFace 为主） |
| 视频 | 基础录制能力（720p/1080p，无时长的简单录制） |
| 编辑 | AI 一键优化；精准局部美颜；编辑体验优化 |

**里程碑验收（M1）**：
- [ ] 帧同步美妆 P0 指标全部达标
- [ ] 视频录制不崩溃、不丢帧（基础美颜）
- [ ] 编辑保存率 ≥ 40%

### Phase 2（中期，4-8 周）：智能体验升级

**目标**：从「能用」到「好用」，降低使用门槛，提升用户粘性。

| 产品线 | 关键交付 |
|--------|----------|
| 拍照 | 智能场景模板（5 个预设）；美颜参数记忆 |
| 视频 | 实时美颜录制（1080p@30fps）；分段录制 |
| 编辑 | 智能消除；专业调色面板；批量编辑 |

**里程碑验收（M2）**：
- [ ] 智能场景模板使用率 ≥ 30%
- [ ] 视频美颜录制帧率 ≥ 30fps 不丢帧
- [ ] 编辑保存率 ≥ 50%

### Phase 3（长期，8-16 周）：创作者生态

**目标**：从「好用」到「热爱」，建立创作者心智，验证商业模式。

| 产品线 | 关键交付 |
|--------|----------|
| 拍照 | 证件照模式；连拍优选；RAW 支持 |
| 视频 | 智能运镜模板；Vlog 一键生成；多平台导出 |
| 编辑 | 证件照制作；拼图/排版；高级编辑能力 |

**里程碑验收（M3）**：
- [ ] 视频导出量占 DAU 的 20%+
- [ ] 整体崩溃率 ≤ 0.1%
- [ ] 编辑能力完整度达到商业级水平（效果对标醒图/美图秀秀）

## 5. 技术底座与依赖

### 5.1 三条线共享的技术底座

```
┌─────────────────────────────────────────────┐
│           大美丽引擎（beauty-engine）         │
│  ├─ OpenGL ES + EGL 渲染管线                │
│  ├─ 美颜算法（磨皮/美白/瘦脸/大眼）          │
│  ├─ 妆容算法（唇色/腮红/眉毛）              │
│  ├─ 风格特效（卡通/素描/浮雕...）           │
│  └─ 滤镜系统（ColorMatrix / LUT）           │
└─────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│           人脸检测引擎                        │
│  ├─ InsightFace 2D106（默认首选）            │
│  ├─ MediaPipe 468→106（备选回退）            │
│  └─ 帧同步系统（FrameSyncManager）           │
└─────────────────────────────────────────────┘
```

### 5.2 关键技术依赖

| 依赖项 | 影响范围 | 状态 | 风险 |
|--------|----------|------|------|
| 帧同步美妆系统 | 拍照 + 视频 | 🔄 进行中 | 高（技术难度大） |
| MediaCodec 录制管线 | 视频 | 📋 未启动 | 中 |
| 本地 Inpainting 模型 | 编辑（智能消除）| 📋 未启动 | 中（模型体积） |
| Selfie Segmentation | 拍照（证件照）+ 编辑（抠图）| 📋 未启动 | 低 |

### 5.3 资源分配建议

| 阶段 | 拍照 | 视频 | 编辑 | 说明 |
|------|------|------|------|------|
| Phase 1 | 40% | 30% | 30% | 帧同步是共享基础，优先投入 |
| Phase 2 | 25% | 40% | 35% | 视频是增长重点，加大投入 |
| Phase 3 | 20% | 35% | 45% | 编辑是变现核心，重点打磨 |

## 6. 项目性质说明

### 6.1 研究目标

本项目为 **技术研究项目**，核心目标包括：

| 目标 | 说明 |
|------|------|
| **算法研究** | 验证本地 AI 美颜算法（人脸检测、关键点驱动、实时渲染）的效果上限 |
| **架构验证** | 验证 Agent Team + Spec 驱动开发模式在影像类项目中的可行性 |
| **性能探索** | 探索 GPU 管线在 Android 相机场景中的性能边界与优化策略 |
| **隐私方案** | 验证 100% 本地处理的技术方案，探索隐私保护与 AI 效果的平衡点 |

### 6.2 无收费计划

- 所有功能完全免费开放，无付费墙
- 无广告、无订阅、无内购
- 核心关注技术指标与用户反馈，而非商业转化
- 研究过程中沉淀的方法论与能力可复用于未来商业项目

## 7. 风险与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|----------|
| 帧同步技术延期 | 高 | 视频美颜录制受阻 | 预留 2 周 buffer；如延期先上视频后补帧同步 |
| 视频录制性能不达标 | 中 | 1080p 录制丢帧 | 先保证 720p@30fps，再优化 1080p |
| 智能消除效果差 | 中 | 编辑核心功能口碑崩 | 初期标注「Beta」，逐步迭代模型 |
| 资源分散 | 中 | 三线并行导致质量下降 | 按 Phase 聚焦，单阶段最多两条线并行 |
| 技术深度不足 | 中 | 研究成果缺乏竞争力 | 定期对标行业最新论文与开源方案，保持技术前沿性 |
