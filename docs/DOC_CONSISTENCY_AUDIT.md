# PicMe 文档一致性分析报告

> **状态**：本报告记录的问题已于 2026-05-03 全部修复。以下内容为历史审计记录，供追溯参考。

## 发现的不一致问题

### 🔴 Critical 级别

#### 1. 美颜引擎架构描述矛盾
- **PRODUCT.md §3.1**: "上层仅保留'大美丽 / 兼容模式'两档心智，兼容模式底层由 GPUPixel 链路承接"
- **FEATURES.md §1.3.4.5**: "当前为单引擎架构，仅保留大美丽一条实现链路" 与 "旧兜底引擎已完全移除"
- **矛盾点**: FEATURES.md 在同一文件内先说"单引擎"，后文又出现"兼容模式"切换文案

#### 2. 风格特效支持范围矛盾
- **PRODUCT.md §3.1**: "大美丽引擎和兼容模式均支持"
- **FEATURES.md §1.4.2**: "大美丽引擎专属支持"
- **FEATURES.md §1.4.3**: "风格特效在所有引擎模式下均可用"
- **矛盾点**: FEATURES.md 先写"专属支持"后写"所有引擎可用"

### 🟠 High 级别

#### 3. 人脸检测来源标识不一致
- **PRODUCT.md §3.1**: `MediaPipe` / `InsightFace` / `GPUPixel` / `None`（4种）
- **FEATURES.md §1.3.6**: 只提 `InsightFace` 主链路、`MediaPipe` 备选链路
- **不一致**: FEATURES.md 未提及 GPUPixel 作为检测来源

#### 4. 性能指标数值冲突
- **PRODUCT.md §2.A**: "拍摄延迟 < 50ms"
- **AGENTS.md §3**: "交互反馈 < 100ms，拍摄快门延迟 < 50ms"
- **FEATURES.md §1.3.4.4**: "实时预览延迟 < 100ms"
- **不一致**: 交互反馈延迟在 AGENTS.md 是 <100ms，但 FEATURES.md 实时预览也是 <100ms，两者定义不同但数值相同

#### 5. 术语命名不统一
| 文档 | 使用的术语 |
|------|-----------|
| PRODUCT.md | 大美丽模式、大美丽（自研 OpenGL ES + EGL）、beauty-engine（模块名） |
| FEATURES.md | 大美丽 、Big Beauty（英文）、美颜总开关 |
| AGENTS.md | 大美丽（主引擎） |
| DEVELOPMENT.md | 未提及 |

### 🟡 Medium 级别

#### 6. 文档边界模糊
- **AGENTS.md §2.3**: 明确禁止"在顶层 AGENTS.md 持续堆叠模块级技术细节"
- **实际**: AGENTS.md §6 有大量模块级技术文档索引，可能越界

#### 7. 美颜参数范围不一致
- **PRODUCT.md §3.1**: 瘦脸 -50~+50，大眼 0-100
- **FEATURES.md §1.3.1**: 瘦脸 -50~+50（负值为丰满脸型），大眼 0-100
- **不一致**: 两者描述略有不同，PRODUCT.md 未说明负值含义

#### 8. 引擎切换文案不一致
- **FEATURES.md §1.3.6.3**: "已切换兼容模式"、"已启用兼容美颜模式"
- **PRODUCT.md §3.1**: "兼容模式底层由 GPUPixel 链路承接"
- **不一致**: FEATURES.md 使用"兼容美颜模式"，PRODUCT.md 使用"兼容模式"

#### 9. 拍照 GPU 化时间标注
- **PRODUCT.md §3.1**: "2026-05 进行中"
- **FEATURES.md §1.3.4.5**: "2026-05 进行中"
- **FEATURES.md §4.2.4**: "大美丽 GPU 化(2026-05)"
- **不一致**: 标注格式不统一，有的加空格有的不加

### 🟢 Low 级别

#### 10. 图标命名不一致
- **PRODUCT.md §3.1**: `AutoFixHigh` / `Face` / `ColorLens` / `SelfImprovement`
- **FEATURES.md §1.3.4.1**: 只提"总开关"、"面部精修"、"妆容调节"、"身材管理"
- **不一致**: FEATURES.md 未使用 Material Icon 名称

#### 11. 默认参数值不一致
- **PRODUCT.md §3.1**: 未明确给出默认值
- **FEATURES.md §1.3.1-1.3.3**: 磨皮 35、美白 25、大眼 20、唇色 40、腮红 20、眉毛 15
- **不一致**: PRODUCT.md 作为 SSOT 未包含默认值

#### 12. 隐私描述冗余
- **PRODUCT.md §2.B**: "零云端"、"本地计算"
- **FEATURES.md §1.3.3**: "仅在本地处理，不上传任何数据"
- **AGENTS.md §3**: "所有 AI 处理必须 100% 本地化"
- **不一致**: 三处描述隐私的措辞不统一

#### 13. 模块路径不一致
- **AGENTS.md §6.1**: `app/src/main/java/com/picme/...`
- **实际项目结构**: `app/src/main/java/com/picme/` 或类似
- **待确认**: 路径是否与代码实际一致

#### 14. 技术文档引用格式
- **FEATURES.md**: 使用 `[BIG_BEAUTY_TECH_SPEC.md](BIG_BEAUTY_TECH_SPEC.md)`
- **AGENTS.md §6.2**: 使用 `docs/BIG_BEAUTY_TECH_SPEC.md`
- **不一致**: 引用格式不统一

#### 15. 版本日期标注
- **PRODUCT.md §2.1**: "2026-04"
- **FEATURES.md §文档边界**: "2026-04"
- **AGENTS.md §1.1**: "2026-04"
- **不一致**: 部分章节日期未更新，如拍照 GPU 化是 "2026-05"
