# PicMe 文档修复方案

> **状态**：本方案已于 2026-05-03 被否决。实际执行的是**完全移除 GPUPixel**的单引擎清理方案，而非本方案建议的"保留 GPUPixel 作为兼容模式"。以下内容为被否决的历史方案记录，供追溯参考。

## 修复计划概览

基于 DOC_CONSISTENCY_AUDIT.md 的分析，本次修复聚焦 **P0（Critical）** 和 **P1（High）** 级别的问题。

---

## P0 修复项

### 修复 #1: 统一美颜引擎架构描述

**问题**: FEATURES.md 描述"单引擎架构"与 PRODUCT.md "两档心智"矛盾

**修改文件**: `docs/FEATURES.md`

**定位**: §1.3.4.5 "大美丽 引擎策略与用户体验"

**修改内容**:
```diff
- **默认主引擎**：大美丽（自研 OpenGL ES + EGL）为唯一引擎。
- **引擎收敛现状**：当前为单引擎架构，仅保留大美丽一条实现链路；旧兜底引擎已完全移除。
+ **默认主引擎**：大美丽引擎（自研 OpenGL ES + EGL）为默认主引擎。
+ **兼容回退链路**：兼容模式底层由 GPUPixel 链路承接，作为大美丽引擎的回退校验与兼容验证链路。
+ **引擎收敛现状**：当前代码与文档体系仅保留大美丽与 GPUPixel 两条链路；旧兜底引擎相关实现、状态枚举与文档引用均已清理，不再作为任何设计选项保留。
```

---

### 修复 #2: 统一风格特效支持范围

**问题**: FEATURES.md 先写"专属支持"后写"所有引擎可用"

**修改文件**: `docs/FEATURES.md`

**定位**: §1.4.2 和 §1.4.3

**修改内容**:
```diff
- **风格特效**：大美丽引擎专属支持卡通、素描、浮雕、海报、交叉线等风格特效...
+ **风格特效**：大美丽引擎和兼容模式均支持卡通、素描、浮雕、海报、交叉线等风格特效...
```

删除 §1.4.3 中重复的"风格特效在所有引擎模式下均可用"描述，避免冗余。
