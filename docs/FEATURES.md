# PicMe 功能细节与业务逻辑规范

本文件记录了 PicMe 核心功能的具体实现逻辑、算法阈值及交互细节，作为 `PRODUCT.md` 的补充。

## 1. 智能相机 (Camera) 深度规范

### 1.1 场景识别算法 (Scene Modes)
- **夜景模式 (Night)**：当环境光度值 (Lux) < 10 时建议自动触发。增加曝光补偿 (+1.0 to +2.0)，并锁定 ISO 范围以平衡噪点。
- **月亮模式 (Moon)**：
    - **触发条件**：变焦倍率 > 3.0x 且检测到高对比度圆形发光体。
    - **逻辑**：自动降低曝光值 (EV) 以清晰显示月面纹理，应用特定的锐化滤镜。

### 1.2 OCR 智能服务逻辑
- **触发时机**：在“文档”模式下，检测到矩形边缘且拍摄完成后，立即启动 `OcrUseCase`。
- **Overlay 交互**：
    - 识别结果应以 `BoundingBox` 形式覆盖在预览图上。
    - 底部展示“复制全部”、“翻译”和“字数统计”快捷操作。

### 1.3 拍摄反馈
- **触感反馈**：调用 `Vibrator` 的 `HapticFeedbackConstants.LONG_PRESS` 模拟机械快门震动。
- **黑场处理**：拍摄瞬时，UI 层插入 50ms 的纯黑 Overlay，模拟单反反光板跳动。

## 2. 动态相册 (Gallery) 深度规范

### 2.1 智能聚类逻辑
- **人物分组 (Face Cluster)**：使用 ML Kit 的 Face Detection 提取特征点。当两张脸的特征距离 < 0.4 时判定为同一人。
- **重复照片判定**：
    - **精确重复**：MD5 哈希值完全一致。
    - **相似照片**：感知哈希 (pHash) 汉明距离 < 5。

### 2.2 媒体查看器 (MediaPager)
- **OCR 联动**：长按图片文字区域或点击“扫描”图标，触发全图 OCR 扫描，并进入文字选择模式。
- **流体动效**：图片展开与收起必须跟随手指坐标（Bezier 曲线：0.4, 0.0, 0.2, 1）。
