## 1. AI 对话首页

### 1.1 页面布局

聊天首页是用户打开 App 后的默认页面，采用全屏对话式设计：

```
┌─────────────────────────────┐
│  顶部栏：PicMe Logo + 设置   │
├─────────────────────────────┤
│                             │
│      对话消息列表区域         │
│      (支持文本/图片消息)       │
│                             │
├─────────────────────────────┤
│  [本地模型 ▼]  输入框  [发送] │
│  (下拉切换本地/远程模型)      │
├─────────────────────────────┤
│  [相机] [相册] [编辑] 快捷入口 │
└─────────────────────────────┘
```

### 1.2 模型切换（输入框下拉）

**下拉选项**：
- **本地模型（Qwen3.5-2B）**：绿色圆点标识，端侧推理，无网络依赖，隐私绝对安全
- **远程模型（DeepSeek 等）**：蓝色圆点标识，云端推理，复杂推理能力强

**切换行为**：
- 点击输入框左侧下拉按钮展开选项
- 切换即时生效，当前对话上下文保留
- 切换时显示 Toast 提示："已切换至本地模型" / "已切换至远程模型"
- 本地模型不可用时（未下载/加载失败），自动禁用该选项并提示

**默认策略**：
- 首次安装：默认远程模型（确保最佳体验）
- 已下载本地模型：默认本地模型（强调隐私+速度）
- 无网络时：强制本地模型（如可用）

### 1.3 对话消息类型

| 消息类型 | 发送方 | 展示样式 | 支持操作 |
|----------|--------|----------|----------|
| **文本消息** | 用户/AI | 用户右对齐气泡 / AI 左对齐气泡 | 长按复制 |
| **图片消息** | 用户 | 缩略图 + 点击放大 | 长按保存/分享 |
| **命令执行** | AI | 带状态图标卡片（进行中/成功/失败）| 点击查看详情 |
| **计划预览** | AI | 交互卡片，含确认/取消按钮 | 确认执行/取消 |
| **图片建议** | AI | 图片 + 文字说明 | 点击应用/忽略 |

### 1.4 快捷入口栏

底部固定栏提供三个核心能力入口：

| 入口 | 图标 | 行为 | 返回行为 |
|------|------|------|----------|
| **相机** | 📷 | 跳转 CameraScreen，全屏拍摄 | 拍照后自动返回聊天页，照片作为图片消息发送 |
| **相册** | 🖼️ | 跳转 GalleryScreen，浏览选择 | 选择照片后返回，照片作为图片消息发送 |
| **编辑** | ✏️ | 跳转 EditorScreen，选择图片编辑 | 保存后返回，编辑后图片作为图片消息发送 |

### 1.5 对话持久化

- 所有对话消息自动保存到本地 Room 数据库
- 支持按会话查看历史（当前仅单会话，后续扩展多会话）
- 应用重启后自动恢复最近对话
- 消息上限：单会话 1000 条，自动清理最早消息
- 支持手动清空对话（顶部栏菜单）

## 2. Agent 交互

### 2.1 交互模式

PicMe 提供两种交互模式，用户可自由切换：

| 模式 | 入口 | 适用场景 |
|------|------|----------|
| **对话模式** | 底部输入栏 | 复杂指令、连续调节、探索发现 |
| **快捷模式** | 右侧控制栏（相机/编辑页内） | 高频单操作、快速调节 |

**端侧能力边界说明**：
- 端侧模型（Qwen3.5-2B）胜任：明确单指令（"拍照""磨皮50"）、简单组合指令（"美白30拍照"）、基础闲聊
- 端侧模型局限：复杂多条件（"3秒后切前置再连拍3张"）、深度上下文推理（"再亮一点"）、开放式深度对话
- 超出端侧能力时：自动降级到远程 LLM（非敏感场景）或规则模板兜底

### 2.2 对话模式

**界面布局**
- 输入栏位于屏幕底部，半透明毛玻璃背景
- 点击展开对话面板，显示历史消息
- 支持语音输入（按住说话）和文字输入
- **模型切换下拉**：输入框左侧显示当前模型标识，点击展开下拉选择

**Agent 反馈规范**
- **确认类**：「已为你调高美颜强度」
- **澄清类**：「你想调节磨皮还是美白？」
- **建议类**：「光线较暗，是否开启夜景模式？」
- **错误类**：「抱歉，我没听懂，请再说一次」
- **模型切换提示**：「已切换至本地模型（Qwen3.5-2B）」

**多轮对话示例**
```
用户：调高美颜
Agent：已为你调高美颜强度（磨皮+20）
用户：再高一点
Agent：已将磨皮提升至 50
用户：换一个滤镜
Agent：已切换至「胶片金」滤镜
```

**模型切换对话示例**
```
用户：[切换至本地模型]
Agent：已切换至本地模型（Qwen3.5-2B），当前对话上下文已保留
用户：帮我看看这张照片
Agent：[分析图片内容]
```

### 2.3 支持的意图

| 意图类别 | 示例指令 | Agent 响应 | 执行层级 |
|----------|----------|------------|----------|
| **对话（基础）** | 「你好」「今天怎么样」 | 友好回复 | L2 本地（Qwen3.5-2B）/ L4 远程 |
| **对话（深度）** | 「解释相对论」「写一首诗」 | 详细回复 | L3/L4 远程 |
| **图片分析** | 「帮我看看这张照片」 | 内容描述 + 建议 | L3 远程（复杂）/ L2 本地（简单） |
| **美颜调节（单参数）** | 「磨皮 50」「美白高一点」 | 参数确认 + 实时预览 | L2 本地 |
| **美颜调节（多参数）** | 「瘦脸 30，美白 50」 | 参数确认 + 实时预览 | L2 本地（简单）/ L3 远程（复杂） |
| **滤镜切换** | 「换个冷调」「复古风格」「原图」 | 滤镜名称 + 视觉切换 | L2 本地 |
| **相机控制** | 「拍照」「翻转镜头」「拉近一点」 | 操作执行 + 状态反馈 | L1 缓存 / L2 本地 |
| **组合指令** | 「3秒后拍照」「美白30并拍照」 | 顺序执行 + 状态反馈 | L2 本地（简单组合）/ L3 远程（复杂组合） |
| **参数查询** | 「当前美颜多少？」「有什么滤镜？」 | 数值回复 / 列表展示 | L2 本地 |
| **上下文推理** | 「再高一点」「换一个」 | 基于历史推断意图 | L2 本地（规则兜底）/ L3 远程 |
| **闲聊** | 「今天天气怎么样」「你会什么」 | 友好回复 + 能力引导 | L4 远程 |

### 2.4 语音交互

**语音输入模式**

| 模式 | 触发方式 | 适用场景 |
|------|----------|----------|
| **Push-to-Talk** | 按住麦克风按钮说话 | 精确控制、嘈杂环境 |
| **WakeWord** | 自动检测语音活动 | 免手操作、快速指令 |

**语音处理流程**
```
AudioRecorder → VAD 检测 → ASR 识别 → LLM 解析 → 命令执行
```

**关键参数**
- VAD 阈值：40dB（平衡灵敏度与误触发）
- 最小语音时长：100ms
- 最大片段时长：4000ms
- 静音超时：800ms

**Agent 语音反馈**
- 识别成功：「已为你调高美颜强度」
- 识别失败：「抱歉，我没听清，请再说一次」
- 命令执行：「正在切换至前置摄像头」
- 端侧能力不足：「这个指令有点复杂，让我想想...」（自动切换远程推理）
- 模型切换：「已切换至本地模型，正在使用 Qwen3.5-2B」

### 2.5 体验红线

- **响应延迟**：Agent 回复首字 < 500ms（本地 L2）/ < 1.5s（远程 L3/L4）
- **语音响应**：ASR 识别到命令执行 < 1.5s（本地）/ < 3s（远程）
- **模型切换延迟**：本地↔远程切换 < 200ms，无卡顿感
- **反馈完整**：每个用户输入必须有可见反馈，包括降级提示
- **渐进披露**：复杂能力通过对话引导发现，不一次性罗列
- **降级透明**：端侧能力不足时，明确告知用户正在切换模式（可选）
- **对话恢复**：应用重启后对话历史恢复 < 500ms

## 3. 智能相机

### 3.1 美颜系统

**功能范围**
- 面部精修：磨皮、美白、瘦脸、大眼
- 妆容调节：唇色、腮红、眉毛
- 身材管理：（预留，未实现）

**交互规范**
- **入口**：右上角控制栏「美颜」按钮
- **面板**：半屏高度，底部滑出
- **实时预览**：参数调整即时生效，延迟 < 100ms
- **对比功能**：长按预览区查看原图，松手恢复

**美颜总开关**
- **默认状态**：关闭（进入相机时人脸检测不运行，节省性能）
- **开启方式**：
  - 用户点击美颜面板开关
  - 调节任意美颜参数时自动开启（`resolveNextBeautySettings`）
- **关闭方式**：
  - 用户手动关闭开关
  - 所有美颜参数归零时自动关闭
- **行为差异**：
  - **关闭时**：跳过人脸检测，仅支持滤镜/调色/风格特效
  - **开启时**：启用人脸检测，支持全部功能（含瘦脸/大眼/唇色/腮红）

**参数范围**

| 功能 | 范围 | 默认值 |
|------|------|--------|
| 磨皮 | 0-100 | 35 |
| 美白 | 0-100 | 25 |
| 瘦脸 | -50~+50 | 0 |
| 大眼 | 0-100 | 20 |
| 唇色 | 0-100 | 40 |
| 腮红 | 0-100 | 20 |
| 眉毛 | 0-100 | 15 |

### 3.2 滤镜系统

**色调滤镜**
- 徕卡经典、徕卡鲜艳、徕卡黑白
- 胶片金、胶片富士、复古
- 冷调、暖调

**风格特效（GPU 实时渲染）**
- 卡通、素描、海报、浮雕、交叉线

**交互规范**
- 横向滚动列表展示
- 风格特效独立区域，可与色调叠加
- 切换动画 < 200ms

### 3.3 拍摄交互

**快门反馈（三位一体）**
1. 触感：50ms 短震动
2. 音效：快门声同步播放
3. 视觉：50ms 黑场闪烁 + 按钮缩放

**人脸十字星**
- 锁定人脸且设备运动时显示
- 画面稳定后自动淡出
- 避免长期遮挡构图

### 3.4 技术约束

- **人脸检测**：MediaPipe Face Mesh 468→106（默认），MNN 2D106 / NCNN 2D106（备选）。InsightFace ONNX 路径已完全移除
- **渲染引擎**：自研 OpenGL ES 管线（大美丽）
- **拍照处理**：GPU 离屏渲染，预览/输出一致性

## 4. 动态相册

### 4.1 媒体查看器

**交互**
- 双指缩放（1x~4x）
- 缩放态优先平移，禁用翻页
- 回到 1x 后恢复翻页
- 展开/收起跟随手指轨迹

**工具栏**
- 顶部：关闭 | 编辑(图片) | OCR | 信息 | 分享 | 删除
- 半透明深色背景，浅色图标

### 4.2 静态图美颜编辑

**入口**
- 工具栏「编辑」按钮（仅图片）
- 长按图片直接进入

**流程**
1. 异步加载原图 + 人脸检测
2. 显示 BeautySelector 面板（同相机）
3. 拖动滑块调节参数，200ms debounce 刷新预览
4. 保存为新文件，不覆盖原图

### 4.3 OCR 文字提取

**触发**
- 工具栏「提取文字」按钮

**展示**
- 原位浮层（白色卡片 + 黑色遮罩）
- Header：标题 + 字数统计
- Content：可滚动文字
- Footer：复制 / 分享按钮

## 5. 设计系统

### 5.1 HyperOS 风格

- **大圆角**：卡片、按钮统一 28dp+
- **毛玻璃**：实时模糊 + 半透明背景
- **流体动效**：Bezier 曲线 (0.4, 0, 0.2, 1)
- **微交互**：长按触感，点击缩放 0.95x

### 5.2 色彩系统

| 用途 | 色值 | 说明 |
|------|------|------|
| Primary | #00E5FF | 科技蓝，强调操作 |
| Secondary | #FF4081 | 强调粉，特殊提示 |
| 深色背景 | #121212 | 深色模式 |
| 浅色背景 | #FFFFFF | 浅色模式 |

### 5.3 摄影美学

- 真实自然：避免过度美颜，保持肤色自然
- 高级质感：参考徕卡色彩科学
- 情感表达：通过滤镜传达情绪

## 6. 通用规范

### 6.1 国际化 (I18N)

**支持语言**
- 英文 (en)
- 简体中文 (zh-CN)
- 繁体中文 (zh-TW)

**术语对照**

| 简体中文 | 繁体中文 | 英文 |
|----------|----------|------|
| 相册 | 相簿 | Gallery |
| 美颜 | 美顏 | Beauty |
| 滤镜 | 濾鏡 | Filter |
| 设置 | 設定 | Settings |
| 提取文字 | 提取文字 | Extract Text |

**规范**
- 字符串资源 ID：`[feature]_[description]` 小驼峰
- 新增功能必须同步三语
- 禁止硬编码用户可见文案

### 6.2 性能要求

| 指标 | 目标 | 验证方式 |
|------|------|----------|
| 冷启动 | < 500ms | 计时器 |
| 快门延迟 | < 50ms | 高速摄像 |
| 参数跟手 | < 100ms | 人工感知 |
| 相册滚动 | 120fps | Systrace |
| GPU 处理 | 1080p < 300ms | 日志 |

### 6.3 隐私红线

- 所有 AI 处理 100% 端侧
- 隐私敏感数据强制本地推理
- 用户数据仅存储在本地
- 权限申请透明，拒绝后优雅降级
