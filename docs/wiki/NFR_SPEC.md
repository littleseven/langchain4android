# PicMe 非功能性需求规格说明书（NFR Spec）

**版本**：1.0  
**状态**：生效中  
**最后更新**：2026-05-14  
**维护者**：PM / QA

## 1. 目的

本文档统一收口 PicMe 所有**可量化的非功能性需求**，作为自动化测试和 CI 门禁的通过/失败标准。所有指标均为**硬红线**（Hard Limit），未达标视为阻塞发布项。

## 7. 验收与门禁

### 7.1 CI 门禁（Merge Blocker）

以下指标未达标时，PR 禁止合并：

- [ ] 单元测试通过率 100%
- [ ] 核心模块单元测试覆盖率 ≥ 70%
- [ ] Lint / detekt 无 ERROR
- [ ] ArchUnit 架构检查通过（`api/` 不依赖 `egl/`）
- [ ] 基准性能测试无退化（启动时间、单帧处理耗时）

### 7.2 发布门禁（Release Blocker）

以下指标未达标时，禁止发布：

- [ ] 整体崩溃率 ≤ 0.1%
- [ ] 美颜引擎崩溃率 ≤ 0.05%
- [ ] ANR 率 ≤ 0.1%
- [ ] 启动首帧 ≤ 500ms
- [ ] 预览帧率 ≥ 30fps（低端机）
- [ ] 帧同步 P0 验收全部通过
