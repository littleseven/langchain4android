Agent Runtime 已搭好"上层建筑"，但支撑语义搜索的"经济基础"——本地多维度索引层——目前几乎空白。智能相册本地索引建设缺失的8个维度
|   优先级  | 缺失维度          | 核心问题                                                      |
| :----: | :------------ | :-------------------------------------------------------- |
| **P0** | **元数据持久化层**   | 无超越 `MediaStore` 的本地数据库，无法支撑人脸/OCR/标签的关联查询                |
| **P0** | **增量索引监听**    | 无 `ContentObserver` + 后台任务队列，新图/删图无法实时同步索引                |
| **P1** | **人脸聚类索引**    | 仅有检测（关键点），缺嵌入向量提取、DBSCAN聚类、`photo_id↔person_id`关联表        |
| **P1** | **OCR倒排索引**   | 仅有内存级OCR，缺后台预提取与`word→photo_id`倒排表，无法搜文字                  |
| **P1** | **场景标签索引**    | 无端侧图像分类模型（如MobileViT），缺`photo_tags`标签表                    |
| **P1** | **地理逆编码索引**   | 仅有原始GPS坐标，缺经纬度→省市区POI的层级映射表                               |
| **P2** | **缩略图加载策略**   | 未提及Glide/Coil或自研缓存，万级图库滑动性能无保障                            |
| **P2** | **Agent搜索桥接** | 缺`GallerySearchCapability` + `QueryBuilder`，LLM意图无法转SQL查询 |
