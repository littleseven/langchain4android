【工作流程】
启动唤醒词监听
    ↓
检测 KWS 引擎
    ├─ 可用 → 启动 KWS (低功耗)
    │   ├─ 关键词检测
    │   ├─ 触发回调
    │   └─ 启动 ASR 进行完整转录
    └─ 不可用 → 降级到 VAD + ASR (原有模式)
    ↓
LLM 解析命令
    ↓
执行相应操作

🚀 启用 KWS 的最小代码改动

【之前】（不使用 KWS）
    val coordinator = VoiceCommandCoordinator(
        asrEngine = asrEngine,
        aiAgentUseCase = aiAgentUseCase,
        onCommand = { command -> handleCommand(command) },
        scope = viewModelScope,
        context = context
    )

【之后】（启用 KWS）
    val coordinator = VoiceCommandCoordinator(
        asrEngine = asrEngine,
        aiAgentUseCase = aiAgentUseCase,
        onCommand = { command -> handleCommand(command) },
        scope = viewModelScope,
        kwsEngine = keywordSpotterEngine,  // ← 仅需添加这一行！
        context = context
    )

✨ 关键点：只需添加一个参数，无需改动任何其他代码！
