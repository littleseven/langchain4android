# 技术路线调整总结（2026-03-29）

## 📋 决策背景

### 原技术方案（R 计划）
- **技术路线**：离屏渲染 + 手动 EGL 管理
- **实施状态**：遇到 CameraX SurfaceProvider 机制限制
- **核心问题**：无法实现离屏渲染到自定义 SurfaceTexture
- **预计周期**：2-3 个月自主研发

### 新双轨策略
基于项目进度和技术风险考虑，决定采用**双轨并行策略**：

```
短期（1-2 周）          中期（1-2 月）           中长期（2-3 月）
    ↓                      ↓                        ↓
PixelFreeEffects      同时运行              R 计划自主研发
SDK 接入            → 积累数据            → 完全替代 SDK
- 快速上线           - 性能监控            - 技术可控
- 验证产品           - Shader 优化         - 定制化能力
- 用户反馈           - 算法迭代            - 零授权成本
```

**实施周期**：1-2 周

**参考文档**：`docs/PIXELFREE_INTEGRATION.md`

### 中长期自主方案（规划中）

**方案名称**：R 计划（自主研发）  
**实施周期**：2-3 个月  
**技术复杂度**：⭐⭐⭐⭐⭐  
**风险等级**：中  

**技术借鉴方向**：
R 计划将借鉴 PixelFreeEffects 的以下技术方案：

1. **渲染架构**
   - GLSurfaceView + Renderer 模式
   - 纹理处理流程（Texture Input → Process → Texture Output）
   - 参数调节接口设计（类似 PFBeautyFilterType 枚举）

2. **美颜算法结构**
   - 磨皮：盒式模糊 + 亮度提升
   - 美白：RGB 通道调整
   - 瘦脸：Vertex Shader 形变

3. **性能优化**
   - 离屏渲染（Pbuffer Surface）
   - EGL 上下文共享
   - 渲染线程优先级（MAX_PRIORITY）

**自主替代目标**：
- ✅ **零授权成本**：无需购买商业 SDK
- ✅ **完全可控**：可定制化开发特殊效果
- ✅ **技术积累**：构建团队核心技术能力
- ⚠️ **时间周期**：预计需要 2-3 个月研发

**参考文档**：`docs/R_PLAN_TECHNICAL_SPEC.md`

## 💡 关键洞察

### 为什么选择双轨策略？

1. **时间窗口**：产品需要快速上线验证市场
2. **技术风险**：R 计划难度高，存在失败风险
3. **资源优化**：短期用 SDK，长期自研，平衡风险与收益
4. **数据驱动**：通过 SDK 积累数据，指导自研方向

### 技术借鉴的价值

PixelFreeEffects 作为成熟的商业 SDK，其技术方案经过市场验证，值得借鉴：

- **架构设计**：GLSurfaceView + Renderer 模式成熟稳定
- **参数设计**：PFBeautyFilterType 枚举覆盖全面
- **性能优化**：离屏渲染、EGL 共享等技术可直接学习
- **算法结构**：磨皮、美白等基础算法具有参考价值

### 自主可控的必要性

虽然 SDK 可以快速上线，但长期来看：

- **成本控制**：商业 SDK 授权费用高昂
- **定制化需求**：特殊效果需要深度定制
- **技术壁垒**：核心技术是团队竞争力
- **数据安全**：完全本地化处理更可靠

## 📌 总结

本次技术路线调整的核心是**平衡短期交付与长期发展**：

- **短期**：务实地使用成熟 SDK，确保产品快速上线
- **中期**：数据驱动，为自主研发积累经验和数据
- **长期**：坚持自主创新，构建核心技术能力

这是一个**进可攻、退可守**的策略：
- 如果 R 计划顺利，可以按时替代 SDK
- 如果 R 计划困难，可以继续使用 SDK
- 无论如何，都能保证产品的持续迭代
