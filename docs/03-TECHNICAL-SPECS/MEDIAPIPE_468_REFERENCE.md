## 对称规则说明

MediaPipe Face Mesh 的面部关键点关于**面部中轴线**（垂直中心线）对称。

- **左侧点**：位于画面左侧（被摄者右脸）
- **右侧点**：位于画面右侧（被摄者左脸）
- **中心点**：位于面部中轴线上，无对称点

**对称规律**：大多数对称对满足 `左侧索引 + 右侧索引 ≈ 500`，但存在例外。

## 10. 对称性验证方法

### 10.1 几何坐标验证

```kotlin
// 检查两个点是否关于面部中轴线对称
fun isSymmetric(leftPoint: Pair<Float, Float>, rightPoint: Pair<Float, Float>): Boolean {
    val centerX = 0.5f // 面部中轴线x坐标
    val tolerance = 0.02f // 容差

    // x坐标应该关于中心对称
    val xSymmetric = Math.abs((leftPoint.first + rightPoint.first) - 1.0f) < tolerance

    // y坐标应该接近
    val ySimilar = Math.abs(leftPoint.second - rightPoint.second) < tolerance

    return xSymmetric && ySimilar
}
```

### 10.2 索引规律验证

```kotlin
// 大多数对称对满足：左侧索引 + 右侧索引 ≈ 500
fun checkSymmetryIndex(leftIndex: Int, rightIndex: Int): Boolean {
    val sum = leftIndex + rightIndex
    return sum in 490..510 // 允许±10的误差
}
```

## 11. 注意事项

1. **坐标系定义**：
   - MediaPipe 返回的坐标为归一化坐标（0-1）
   - 原点(0,0)在左上角
   - x轴向右，y轴向下

2. **前置摄像头镜像**：
   - 使用前置摄像头时，需要水平镜像：`x = 1 - x`
   - 镜像后，"左侧"和"右侧"互换

3. **对称精度**：
   - 理想情况下，对称点的y坐标应该完全相同
   - x坐标应该关于0.5对称（即 `x_left + x_right = 1.0`）
   - 实际检测中存在微小偏差是正常的

4. **中心点处理**：
   - 位于面部中轴线上的点没有对称点
   - 包括：鼻尖(4)、鼻梁(6, 168, 195, 197)、唇珠(0)等

5. **特殊情况**：
   - 某些对称对不严格遵循 `left + right = 500` 规律
   - 建议通过实际坐标验证对称性

6. **轮廓差异**：
   - MediaPipe FACE_OVAL 为36点闭合曲线（含额头）
   - 106点标准为33点开放曲线（不含额头）
   - 映射时需要选择不含额头的 MediaPipe 轮廓点

7. **瞳孔检测**：
   - 需要开启 `refine_landmarks=True`
   - 瞳孔点索引为 468-475（478点模式）

8. **左右定义**：
   - MediaPipe 中的"左/右"是基于**观察者视角**，与106点标准一致

9. **深度信息**：
   - MediaPipe 每个点包含 (x, y, z) 三维坐标，z轴以鼻尖为原点
