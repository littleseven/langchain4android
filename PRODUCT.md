# PicMe 产品需求规格说明书 (SSOT)

## 1. 愿景与使命
**愿景**：成为 Android 平台上最快、最私密、审美最纯粹的相机与相册应用。
**使命**：提供零延迟的拍摄体验和基于本地 AI 的媒体管理，确保所有数据 100% 留在设备上。

## 2. 核心支柱与指标 [严格执行]

### A. 性能 (零延迟)
- **启动速度**：冷启动至预览界面 < 500ms。**严禁使用启动页 (Splash Screen)**。
- **滑动性能**：在 1000+ 条目的相册中保持 120fps 满帧滚动。
- **拍摄延迟**：快门延迟 < 50ms。必须调用 `ImageCapture.CAPTURE_MODE_MINIMIZE_LATENCY`。

### B. 隐私 (本地 AI)
- **零云端**：AI 模块（人脸、OCR、分类）严禁申请或使用网络权限。
- **本地计算**：所有分析任务必须通过 ML Kit、Media3 和 Room 在本地完成。

### C. 审美 (HyperOS 风格)
- **视觉语言**：大圆角 (28dp+), 实时高斯模糊, 非线性流体动效。
- **交互逻辑**：手势优先（如：下滑关闭、长按多选、缩放切换视图）。

## 3. 功能详细规范

### 3.1 智能相机
- **OCR 服务**：集成于拍摄流中的后台服务，而非独立页面。
    - **文档模式**：拍摄后自动静默触发 OCR；识别结果通过 `TextOverlay` 透明层在预览图上实时渲染。
- **交互反馈**：快门操作必须同步触发：**触感 (Impact) + 音效 (Shutter) + 视觉 (Blackout)** 三位一体反馈。
- **智能对焦**：强制开启人脸优先测光与连续自动对焦。

### 3.2 动态相册
- **分组逻辑**：
    - 基础：按时间轴线线性排列。
    - 智能分组：人物（人脸聚类）、场景标签、文档。
- **即时性**：进入智能分组时，必须秒级过滤非相关媒体。
- **OCR 集成**：
    - **入口**：媒体详情页 (MediaPager) 提供“提取文字”按钮。
    - **快捷操作 [NEW]**：长按图片文字区域触发 OCR 识别。
    - **交互逻辑 [NEW]**：触发时同步执行 `LongPress` 触感反馈（模拟快门）。
    - **展示方式**：识别结果通过原位浮层 (Overlay) 展示，支持一键复制与分享，无需跳转独立页面。

## 4. 设计与 UX 准则
- **排版系统**：使用系统默认字体，通过字重 (Weight) 区分层级，而非字号。
- **色彩规范**：极简单色调，仅对关键 Action 使用高对比度强调色。
- **无障碍**：所有 UI 元素必须具备完整的 `contentDescription`。

## 5. Agent 执行约束
- **[MUST]** 任何 UI 状态变更必须产生 `PicMe:UX` 格式的结构化日志。
- **[NEVER]** 严禁实现任何依赖云端连接或第三方非私有化 API 的功能。
- **[I18N]** 所有文案必须同步支持 英文、简体中文、繁体中文。
