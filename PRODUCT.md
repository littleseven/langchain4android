# PicMe 产品需求规格说明书 (SSOT)

## 1. 愿景与使命
**愿景**：成为 Android 平台上最快、最私密、审美最纯粹的相机与相册应用。
**使命**：提供零延迟的拍摄体验和基于本地 AI 的媒体管理，确保所有数据 100% 留在设备上。

## 2. 核心支柱与指标 [严格执行]

### 2.1 文档承接约定（2026-04）
- 本文档定义产品目标、关键指标与验收口径（What）。
- 交互流程与体验规则统一承接到 `docs/FEATURES.md`（How）。
- 模块实现细节、代码规范与检查清单统一承接到模块 `AGENTS.md` 与 `docs/AGENTS_SPEC.md`（Implementation）。

### A. 性能 (零延迟)
- **启动速度**：冷启动至预览界面 < 500ms。**严禁使用启动页 (Splash Screen)**。
- **滑动性能**：在 1000+ 条目的相册中保持 120fps 满帧滚动。
- **拍摄延迟**：快门延迟 < 50ms。必须调用 `ImageCapture.CAPTURE_MODE_MINIMIZE_LATENCY`。

### B. 隐私 (本地 AI)
- **零云端**：AI 模块（人脸、OCR、分类）严禁申请或使用网络权限。
- **本地计算**：所有分析任务必须通过 ML Kit、Media3 和 Room 在本地完成。

### C. 审美 (HyperOS 风格)
- **视觉语言**：大圆角 (28dp+), 实时高斯模糊, 非线性流体动效。
- **交互逻辑**：手势优先（如：下滑关闭、长按多选、缩放切换视图）。

## 3. 功能详细规范

### 3.1 智能相机 (Camera)
- **场景识别能力**：
    - **夜景模式 (Night)**：环境光度值 (Lux) < 10 时自动触发，增加曝光补偿 (+1.0 to +2.0)
    - **月亮模式 (Moon)**：变焦倍率 > 3.0x 且检测到高对比度圆形发光体时触发，自动降低曝光值
- **拍摄交互与反馈**：
    - **三位一体反馈**：快门操作必须同步触发触感 (Impact) + 音效 (Shutter) + 视觉 (Blackout)
    - **触感反馈**：调用 `HapticFeedbackConstants.LONG_PRESS` 模拟机械快门震动（时长 50ms）
    - **黑场闪烁**：拍摄瞬时插入 50ms 纯黑 Overlay，模拟单反反光板跳动
    - **视觉确认**：快门按钮按下时缩放至 0.95x，释放后恢复
    - **人脸锁定十字星**：仅在“已锁定人脸且设备处于运动”阶段显示；当人脸持续锁定且画面静止（稳定）超过短暂阈值后，执行淡出并隐藏，避免遮挡构图
- **美颜系统**：
    - **入口层级**：拍摄页右上角一级入口，采用“总开关 + 三类子功能”结构。
    - **图标与顺序**（由上到下）：
        - **美颜总开关** (`AutoFixHigh`)：统一启停美颜链路。
        - **面部精修** (`Face`)：磨皮、美白、瘦脸、大眼。
        - **妆容调节** (`ColorLens`)：唇色、腮红、眉毛。
        - **身材管理** (`SelfImprovement`)：丰胸、长腿。
    - **信息架构分组**：美颜组后依次是构图组（画幅 -> 网格）与风格组（场景 -> 滤镜），组间增加弱分隔留白。

    - **交互规范**：
        - **面板设计**：半屏高度 (屏幕高度的 50%)，底部滑出，符合 HyperOS 风格
        - **互斥逻辑**：三个子功能面板互斥，打开一个自动关闭其他
        - **关闭方式**：点击面板外部遮罩、点击关闭按钮、或点击其他功能按钮
        - **实时预览**：参数调整实时生效，延迟 < 100ms
        - **参数范围**：磨皮/美白/大眼 0-100，瘦脸 -50~+50
        - **总开关联动**：用户调整任一参数时自动开启美颜；全部参数归零时自动关闭美颜
    - **功能详情**：
        - **面部精修**：
            - **磨皮 (Smoothing)**：基于双边滤波的肤色平滑，保留边缘细节，强度 0-100
            - **美白 (Whitening)**：智能色调提升，均匀肤色，强度 0-100
            - **瘦脸 (Slim Face)**：基于人脸 landmarks 的脸型重塑，安全范围 -50~+50
            - **大眼 (Big Eyes)**：眼睛区域放大，保持自然比例，强度 0-100
        - **妆容调节**：
            - **唇色 (Lip Color)**：智能唇部识别，自然上色，支持 12 种色号选择
            - **腮红 (Blush)**：苹果肌区域自然红润，强度 0-100
            - **区域策略**：双颊中心默认外移并上提，避开鼻翼与嘴角；鼻梁中线保留留白，防止双颊重叠
            - **眉毛 (Eyebrow)**：眉形加深与填充，强度 0-100
        - **身材管理**：
            - **丰胸 (Body Enhancement)**：基于人体关键点的安全调整，范围 -30~+30
            - **长腿 (Leg Extension)**：视觉比例优化，保持身体协调，范围 0-50
    - **大美丽 产品策略**：
        - **默认主引擎**：大美丽（自研 OpenGL ES + EGL）作为实时美颜默认引擎。
        - **稳定兜底**：保留 PixelFreeEffects 作为备用链路，确保核心拍摄流程可用。
        - **双引擎共存**：两套引擎长期共存，并提供设置页切换能力。
        - **自动回退**：当 大美丽初始化失败或运行异常时，自动切换至 PixelFreeEffects，并提示“已切换稳定模式”。
        - **回退可恢复**：异常恢复后允许用户在设置页重新切回 大美丽。
        - **长期库化目标**：大美丽 持续演进为独立视觉能力基础库（类似 PixelFree 的产品形态），App 作为能力消费者。
        - **能力边界**：基础库统一承载美颜、滤镜、妆容能力，对外提供稳定 API 与版本，不暴露底层渲染细节。

    - **调试与性能可观测性**：
        - **调试总开关**：设置页默认开启；统一控制拍摄页调试浮层、调试工具入口和 Log 入口显隐
        - **性能指标**：调试浮层实时展示 FPS、处理耗时、预览延迟、CPU 占用、空帧计数
        - **人脸检测模式**：默认 `Landmark`（性能优先），可切换 `Contour`（精度优先）
        - **动态检测间隔**：默认开启，可在 280~450ms 区间自适应调整
        - **强度档位**：支持保守/平衡/激进三档，默认平衡
- **滤镜系统**：
    - **LEICA_CLASSIC**：强调暗部细节，低饱和度，轻微提高对比度（街头摄影、建筑、纪实）
    - **FILM_GOLD**：暖色调，模拟柯达胶片质感，高光柔和（人像、旅行、美食）
    - **COOL**：冷色调，清爽干净，轻微提高饱和度（风景、城市夜景）
    - **WARM**：暖色调，温馨氛围，肤色自然（日落、室内、亲子）
    - **性能要求**：实时预览流畅，内存占用合理，切换动画 < 200ms
- **OCR 智能服务**：
    - **定位**：无感后台服务，集成于拍摄流与相册查看器
    - **触发时机**：拍摄中检测到矩形边缘且拍摄完成后；相册内点击工具栏"提取文字"或长按文字区域
    - **交互展示**：原位浮层 (Overlay) 展示，白色背景 +70% 黑色遮罩，大圆角设计 (28dp+)
    - **卡片结构**：Header 区（标题 + 字数统计）+ Divider + Content 区（可滚动文字）+ Footer 区（复制/分享按钮）
    - **功能闭环**：一键复制（剪贴板 + 触感 + Toast）、一键分享（系统 Intent）、快速关闭（点击外部或关闭按钮）
- **实现细节**：见 `docs/FEATURES.md Section 1`

### 3.2 动态相册 (Gallery)
- **分组逻辑**：
    - **基础分组**：按时间轴线性排列
    - **智能分组**：
        - **人物分组 (Face Cluster)**：使用 ML Kit Face Detection，特征距离 < 0.4 判定为同一人
        - **场景标签**：基于 AI 识别的场景分类
        - **文档分类**：自动识别并归类文档类图片
- **重复照片判定**：
    - **精确重复**：MD5 哈希值完全一致
    - **相似照片**：感知哈希 (pHash) 汉明距离 < 5
- **媒体查看器 (MediaPager)**：
    - **流体动效**：图片展开与收起跟随手指坐标（Bezier 曲线：0.4, 0.0, 0.2, 1）
    - **交互细节**：支持双指缩放（1.0x~4.0x）与缩放后拖拽；缩放态优先图片平移并临时禁用翻页，恢复至 1x 后自动恢复左右翻页
    - **工具栏布局**：顶部安全区域下方 16dp，左侧关闭按钮，右侧 OCR、信息、分享、删除按钮（间距 8dp）
    - **视觉样式**：半透明黑色背景 (alpha 0.5)，白色图标，选中状态 Primary 色高亮
- **批量操作（相册列表）**：
    - **批量选择**：支持长按进入选择模式与连续滑动批量选择
    - **批量分享**：选择模式下支持一键批量分享，单张走 `ACTION_SEND`，多张走 `ACTION_SEND_MULTIPLE`
- **OCR 集成**：
    - **入口**：媒体详情页 (MediaPager) 右上角工具栏"提取文字"按钮
    - **快捷操作**：长按图片文字区域触发 OCR 识别，同步执行 `LongPress` 触感反馈
    - **交互逻辑**：触发时同步执行 `LongPress` 触感反馈（模拟快门）
    - **展示方式**：识别结果通过原位浮层 (Overlay) 展示
        - **一键复制**：点击按钮调用系统剪贴板，并触发 `Impact` 触感反馈及 Toast 提示
        - **一键分享**：点击按钮调用 Android 系统原生分享 (Intent.ACTION_SEND)，支持分享至笔记、社交应用
        - **快速关闭**：点击右上角关闭按钮或点击背景遮罩区域即可退出
- **实现细节**：见 `docs/FEATURES.md Section 2`；实现规范见模块 `AGENTS.md` 与 `docs/AGENTS_SPEC.md`

### 3.3 设计系统与规范
- **HyperOS 视觉风格**：
    - **大圆角设计**：所有卡片、按钮统一使用 28dp+ 圆角
    - **毛玻璃效果**：实时高斯模糊 (blur 20dp) + 半透明背景 (alpha 0.5-0.8)
    - **流体动效**：Bezier 曲线 (0.4, 0.0, 0.2, 1)，模拟物理世界惯性
    - **微交互反馈**：长按触发触感反馈，点击触发缩放反馈 (0.95x)
- **色彩系统**：
    - **Primary 色**：科技蓝 (#00E5FF) - 用于强调操作和选中状态
    - **Secondary 色**：强调粉 (#FFFF4081) - 用于特殊提示和警告
    - **深色模式**：背景 #121212，表面 #1E1E1E，文字白色
    - **浅色模式**：背景白色，表面浅灰，文字黑色
- **摄影美学原则**：
    - **真实自然**：避免过度美颜和失真，保持人物肤色自然
    - **高级质感**：参考徕卡色彩科学，强调暗部细节和层次感
    - **情感表达**：通过滤镜传达不同情绪（温暖、冷静、怀旧）
    - **专业调色**：提供精细的参数控制（饱和度、对比度、色温）
- **实现细节**：见 `docs/FEATURES.md Section 3`

### 3.4 通用规范

#### 3.4.1 国际化 (I18N)
- **支持语言**：英文、简体中文、繁体中文
- **术语规范**：
    - 简体 (CN)：相册、滤镜、设置、拍照
    - 繁体 (TW/HK)：相簿、濾鏡、設定、拍照
    - 英文 (EN)：Gallery, Filters, Settings, Camera/Capture
- **多语言词汇表**：完整词汇表见 `docs/FEATURES.md Section 4.1.1`
- **实现规范**：
    - 资源文件位置：各语言文案定义在 `res/values-*/strings.xml` 中
    - 命名规则：字符串资源 ID 采用小驼峰命名，格式为 `[feature]_[description]`
    - 占位符处理：使用标准占位符，确保各语言版本语义通顺
    - 复数形式：英文需支持单复数变化，中文使用量词区分
    - 文案更新流程：新增功能时，必须同步更新英文、简体中文、繁体中文三个版本
- **实现细节**：见 `docs/FEATURES.md Section 4.1`

#### 3.4.2 启动与性能优化
- **冷启动流程 (< 500ms)**：
    - **时间分配**：
        - 0-50ms：Application 初始化（仅注册必要组件）
        - 50-150ms：权限检查与请求（相机、存储）
        - 150-300ms：CameraX 初始化并绑定生命周期
        - 300-500ms：首次预览帧渲染完成
    - **禁止事项**：
        - ❌ 显示启动页 (Splash Screen)
        - ❌ 执行数据库迁移或大量数据加载
        - ❌ 等待 AI 模型加载完成（应后台异步加载）
- **相册滚动优化 (120fps)**：
    - **用户体验目标**：
        - 在包含 1000+ 张照片的相册中滑动时，保持视觉流畅无卡顿
        - 缩略图加载应跟手，无明显白屏或延迟
        - 快速滑动停止后，首张图片应立即清晰显示
    - **加载策略**：
        - 优先保证当前可见区域的图片清晰度
        - 预加载用户即将看到的图片（前后相邻项）
        - 内存紧张时自动降低非可见图片的优先级
- **拍摄延迟优化 (< 50ms)**：
    - **用户体验目标**：
        - 按下快门后应立即听到拍照音效和触感反馈（感知延迟 < 50ms）
        - 照片保存过程不应阻塞界面，用户可立即查看下一张照片
        - 整个拍摄流程（按下到可继续拍摄）应在 200ms 内完成
    - **反馈机制**：
        - 必须同时提供三种反馈：触感震动、快门音效、屏幕黑场闪烁
        - 任一反馈的缺失都会导致用户感知"卡顿"
- **实现细节**：见 `docs/FEATURES.md Section 4.2`

#### 3.4.3 隐私与安全
- **本地 AI 约束**：
    - **隐私承诺**：
        - 所有 AI 功能（人脸检测、OCR 识别、智能分类）必须在设备上完成
        - 严禁将用户照片上传至云端进行分析
        - 应用不应具备联网能力（不申请网络权限）
    - **用户体验影响**：
        - 离线环境下所有功能必须正常使用
        - AI 处理速度应达到秒级响应（人脸检测 < 1s，OCR < 3s）
        - 模型精度可能略低于云端方案，但换取了极致的隐私保护
- **数据存储安全**：
    - **用户数据保护**：
        - 用户的照片、相册分组等核心数据必须存储在设备本地
        - 支持用户设置应用锁（密码或生物识别）保护隐私
        - 提供"隐藏相册"功能，允许用户对敏感内容二次加密
    - **权限透明**：
        - 仅在必要时向用户申请权限（如首次访问相册时）
        - 明确告知用户每个权限的用途（相机、存储、人脸）
        - 用户拒绝权限后，应降级功能而非崩溃（如无存储权限时仅能拍照不能保存）
- **实现细节**：见 `docs/FEATURES.md Section 4.3`

## 4. 设计与 UX 准则
- **排版系统**：使用系统默认字体，通过字重 (Weight) 区分层级，而非字号。
- **色彩规范**：极简单色调，仅对关键 Action 使用高对比度强调色。
- **无障碍**：所有 UI 元素必须具备完整的 `contentDescription`。

## 5. Agent 执行约束
- **[MUST]** 任何 UI 状态变更必须产生 `PicMe:UX` 格式的结构化日志。
- **[NEVER]** 严禁实现任何依赖云端连接或第三方非私有化 API 的功能。
- **[I18N]** 所有文案必须同步支持 英文、简体中文、繁体中文。
